

# Key concepts in vector search

Before you explore how vector search works, review these key concepts.

## Embeddings

An embedding is a numerical representation of data. It is a vector of floating-point numbers, typically ranging from 128 to 4,096 dimensions depending on the model.

Embeddings capture meaning. Items with similar meaning are located close together in the vector space, while unrelated items are farther apart. For example, in a text embedding model, the vectors for "Paris" and "France" are close together, as are "Berlin" and "Germany."

For example, the query "a quick brown fox" can be represented as:

```
[ 0.231, -0.445,  0.612,  0.178, -0.334,  0.789,
  0.056, -0.223,  0.501,  0.367, -0.612,  0.144,
  0.788, -0.091,  0.423, -0.317]
```

_16-dimensional sentence embedding (real models typically use 768 to 3,072 dimensions)_

Using the cosine similarity to nearest neighbors, the system might return results like:

| Text                | Similarity |
|---------------------|------------|
| "the fast red fox"  | 0.88       |
| "a speedy brown dog"| 0.81       |
| "brown animals run" | 0.76       |
| "stock market crash"| 0.03       |

## Embedding models

Embeddings are generated by machine learning models, typically transformer-based neural networks trained on large datasets. Different models are designed for different data types and use cases.

| Model type | Description and examples |
|---|---|
| **Text** | Encodes sentences, paragraphs, or documents. Examples include OpenAI text-embedding-3, Cohere Embed, and Sentence-BERT. |
| **Image** | Encodes visual content into vectors. Examples include CLIP and ResNet-based models. |
| **Multimodal** | Encodes text and images into a shared space, enabling cross-modal search. Examples include CLIP and ImageBind. |
| **Code** | Encodes source code and programming structures. Examples include CodeBERT and GitHub Copilot embeddings. |
| **Audio** | Encodes speech or music. Examples include wav2vec 2.0 and CLAP. |

## Vector databases

A vector database is a storage system designed to store and query large collections of vectors efficiently. Unlike relational databases, which are optimized for exact matches, vector databases are optimized for similarity search.

Common vector databases and libraries include:

- **Pinecone** — Managed vector database  
- **Weaviate** — Open-source with built-in vectorization and hybrid search  
- **Qdrant** — Open-source, focused on performance  
- **Milvus** — Distributed and designed for large-scale workloads  
- **pgvector** — PostgreSQL extension for vector search  
- **FAISS** — Library for fast similarity search (not a database)  
- **Chroma** — Lightweight and commonly used in RAG applications  

### Approximate nearest neighbor (ANN) algorithms

The goal of nearest neighbor search is to find the data point most similar to a query point.

In high-dimensional spaces, such as vector embeddings used in AI systems, exact search is computationally expensive and slow.

Approximate nearest neighbor (ANN) algorithms improve performance by trading a small amount of accuracy for faster results. They do not guarantee the closest possible match, but they return results that are close enough for most use cases, often in near real time.

| Algorithm | Description and trade-offs |
|---|---|
| **HNSW** | Uses a multi-layer graph structure for fast search. High accuracy and speed, but higher memory usage. |
| **IVF** | Groups vectors into clusters and searches only relevant groups. Efficient, but accuracy depends on configuration. |
| **LSH** | Hashes similar vectors into the same buckets. Very fast, but lower accuracy. |
| **ScaNN** | Uses quantization and optimized distance scoring. Performs well on large datasets. |
| **PQ** | Compresses vectors to reduce memory usage. Often combined with IVF. Reduces accuracy. |

## Similarity metrics

After vectors are stored, the system measures how similar two vectors are using a similarity or distance metric. The choice depends on the model and use case.

| Metric | Description |
|---|---|
| **Cosine similarity** | Measures the angle between vectors. Ignores magnitude. Common for text embeddings. Values range from -1 to 1. |
| **Euclidean distance (L2)** | Measures straight-line distance. Lower values indicate greater similarity. |
| **Dot product** | Fast to compute. Accounts for both direction and magnitude. Often used in RAG systems. |
| **Hamming distance** | Counts differences between binary vectors. Used for compact representations. |
